# AnnSight 文件上传功能说明

**更新日期**: 2026-03-21
**功能状态**: ✅ 已完成

---

## 📁 支持的文件格式

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| 纯文本 | `.txt` | 每行一条数据 |
| CSV 文件 | `.csv` | 每行一条数据 |
| Excel | `.xlsx`, `.xls` | 读取第一列数据（需解析库支持） |
| JSON | `.json` | 支持数组或 `{data:[...]}` 格式 |

---

## 🚀 使用方式

### 方式一：文件上传（推荐）

1. 访问 **源数据管理** 页面
   ```
   http://localhost:3000/raw-data.html
   ```

2. 点击 **"批量上传"** 按钮

3. 切换到 **"📁 文件上传"** 标签

4. 选择文件：
   - **点击选择**: 点击虚线框选择文件
   - **拖拽上传**: 直接拖拽文件到虚线框内

5. 填写信息：
   - 数据来源（知乎/小红书/访谈等）
   - 批次 ID（自动生成，可修改）

6. 点击 **"上传并处理"**

---

### 方式二：文本输入

适合少量数据（< 50 条）：

1. 点击 **"批量上传"** 按钮

2. 保持 **"📝 文本输入"** 标签

3. 直接粘贴文本（每行一条）

4. 点击 **"上传并处理"**

---

## 📋 文件格式示例

### TXT 文件格式
```
职场沟通中，学会倾听比说话更重要
情绪管理是情商的核心组成部分
建立良好的人际关系需要主动倾听
时间管理的四象限法则
```

### CSV 文件格式
```csv
content
职场沟通中，学会倾听比说话更重要
情绪管理是情商的核心组成部分
建立良好的人际关系需要主动倾听
```

### JSON 文件格式
```json
[
  {"content": "职场沟通中，学会倾听比说话更重要"},
  {"content": "情绪管理是情商的核心组成部分"},
  {"content": "建立良好的人际关系需要主动倾听"}
]
```

**支持的 JSON 格式**：
- **数组格式**：`[{content: "..."}, ...]`
- **数据对象**：`{data: [{content: "..."}, ...]}`
- **自动提取**：递归提取所有字符串值

---

## ⚠️ 注意事项

1. **编码格式**: 文件需使用 UTF-8 编码
2. **数据分隔**: TXT/CSV 文件每行一条数据
3. **Excel 支持**: 当前版本 Excel 读取功能有限，建议转换为 TXT/CSV
4. **JSON 格式**: 支持数组或 `{data:[...]}` 格式，自动提取 content/text 字段
5. **文件大小**: 建议单次上传 < 1000 条数据
6. **去重检测**: 上传后自动进行 MD5 去重

---

## 🔧 故障排除

### 问题：上传后显示 0 条数据
**原因**: 文件编码不是 UTF-8
**解决**: 用文本编辑器转换编码为 UTF-8

### 问题：Excel 文件上传失败
**原因**: Excel 需要解析库支持
**解决**: 另存为 CSV 格式后上传

### 问题：数据重复
**原因**: 同一批次重复上传
**解决**: 使用不同的批次 ID

## 📊 上传后处理流程

```
文件上传 → 按行分割 → MD5 去重 → ETL 处理 → 生成加工数据
                                        ↓
                                   L1 清洗 → L2 结构化 → L3 评估
```

### 裂变效果

系统支持 **1:N 裂变**：一条源数据可以生成多条加工数据。

```
1 篇知乎长文 → LLM 分析 → [知识点 1, 知识点 2, 案例 1, 技巧 1]
1 个访谈录音 → 转录文本 → [案例 3, 技巧 2, 教训 1]
```

**示例**：
- 上传 10 条源数据
- LLM 分析后提取出 35 个独立知识点
- 生成 35 条加工数据（裂变系数 3.5x）

**提示词支持**：你可以通过修改 LLM 提示词来控制裂变数量和格式。

---

## 🎯 最佳实践

1. **按来源分类**: 不同来源的数据使用不同的批次 ID
2. **批次命名**: 使用有意义的命名（如：`zhihu-2026-03-21`）
3. **定期上传**: 建议每天固定时间上传当日数据
4. **质量检查**: 上传后检查源数据列表，确认数据正确
