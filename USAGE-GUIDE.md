## 数据上传流程

### 方式一：文件上传（推荐）

**适用场景**: 批量数据（> 10 条）

**支持格式**:
- `.txt` - 纯文本，每行一条数据
- `.csv` - CSV 文件，每行一条数据
- `.xlsx`/`.xls` - Excel 文件（建议转 CSV）

**操作步骤**:

1. **打开上传对话框**
   - 访问 http://localhost:3000/raw-data.html
   - 点击右上角 **"批量上传"** 按钮

2. **选择文件上传标签**
   - 点击 **"📁 文件上传"** 标签页

3. **上传文件**
   - **点击选择**: 点击虚线框选择文件
   - **拖拽上传**: 直接拖拽文件到虚线框内

4. **填写信息**
   - **数据来源**: 选择数据来源（知乎/小红书/访谈/用户投稿）
   - **批次 ID**: 自动生成，可修改为有意义的命名

5. **确认上传**
   - 点击 **"上传并处理"** 按钮
   - 等待处理完成
   - 查看上传结果（成功数/失败数）

**批次命名建议**:
```
zhihu-2026-03-21      # 知乎数据
xiaohongshu-0321      # 小红书数据
interview-user001     # 访谈数据
submission-20260321   # 用户投稿
```

## 裂变功能

### 什么是裂变

一条源数据（种子）可以裂变成多条不同用途的加工数据：

```
1 篇知乎文章 → LLM 分析 → 5 条 RAG 素材 + 3 条微调数据 + 2 条创作素材
```

### 支持的用途

| 用途 | 说明 | 输出格式 |
|------|------|---------|
| 📚 RAG 素材 | 知识库文档 | 知识点、案例、技巧卡片 |
| 🤖 微调数据 | SFT 训练样本 | 多轮对话、问答对 |
| ✍️ 内容创作 | 创作素材 | 选题、大纲、素材片段 |

### 前端操作

1. **访问上传页面**
   - http://localhost:3000/raw-data.html

2. **点击"批量上传"**

3. **选择裂变方向**（多选）
   ```
   ┌─────────────────────────────────┐
   │  裂变方向（多选）                │
   │  ☑ 📚 RAG 素材                  │
   │  ☑ 🤖 微调数据                  │
   │  ☐ ✍️ 内容创作素材              │
   └─────────────────────────────────┘
   ```

4. **上传文件/文本**

5. **查看结果**
   ```
   上传完成：成功 10/10（裂变 35 条加工数据）
   ```

### 工作流程

```
选择用途 → 上传数据 → LLM 裂变分析 → 保存多条数据
```

### 统计筛选

访问统计看板，顶部可按用途筛选：

```
📌 按用途筛选：☑ 📚 RAG 素材  ☑ 🤖 微调数据  ☑ ✍️ 内容创作
```

### 方式二：文本输入

**适用场景**: 少量数据（< 50 条）

**操作步骤**:

1. 点击 **"批量上传"** 按钮
2. 保持 **"📝 文本输入"** 标签页
3. 粘贴文本（每行一条数据）
4. 填写来源和批次 ID
5. 点击 **"上传并处理"**

**文本格式示例**:
```
职场沟通中，学会倾听比说话更重要
情绪管理是情商的核心组成部分
建立良好的人际关系需要主动倾听
```

### 数据去重机制

系统自动进行 **MD5 去重**，防止重复数据进入系统。

- **完全重复**: 相同内容的 MD5 值相同，自动拦截
- **批次内去重**: 同批次相同内容只保留一条
- **全局去重**: 跨批次检测重复内容

## 数据审核流程

### AI 自动处理流程

```
上传数据 → MD5 去重 → L1 清洗 → L2 结构化 → L3 评估
                                              ↓
                                     AI 置信度评分
                                          │
                    ┌─────────────────────┼─────────────────────┐
                    ↓                     ↓                     ↓
            高置信度 (≥0.8)         中置信度 (0.5-0.8)     低置信度 (<0.5)
            → AI 自动通过           → 人工审核            → 人工审核
            → 冷却期 24h            → 优先级中            → 优先级高
```

### 人工抽检

**访问**: http://localhost:3000/spotcheck.html

**抽检策略**: 分层抽样，确保每个类型/分类都有代表样本

**操作步骤**:

1. **加载抽检样本**
   - 点击 **"加载抽检样本"** 按钮
   - 系统自动从 AI 自动通过的数据中抽样

2. **审核样本**
   - 查看 AI 分类结果
   - 判断是否正确
   - 如错误，修正分类/标签

3. **提交修正**
   - 点击 **"提交修正"** 按钮
   - 系统记录修正日志
   - 用于 AI 准确率统计

**AI 修正功能**:
- 发现 AI 系统性偏差时，可使用 **"批量修正"** 功能
- 按条件筛选（如：类型=教训案例，分类=职场）
- 批量修正为目标分类

### 低置信度审核

**访问**: http://localhost:3000/spotcheck.html

**操作步骤**:

1. **加载待审核数据**
   - 点击 **"加载待审核数据"** 按钮
   - 系统加载置信度 < 0.8 的数据

2. **人工审核**
   - 查看 AI 分析结果
   - 判断是否准确
   - 选择 **"通过"** 或 **"拒绝"**

3. **提交审核**
   - 点击 **"提交审核"** 按钮
   - 数据状态更新

## 常见问题

### 上传相关问题

**Q: 上传后显示 0 条数据？**
A: 检查文件编码是否为 UTF-8。用文本编辑器（如 VSCode）打开文件，另存为 UTF-8 编码。

**Q: Excel 文件上传失败？**
A: 当前版本 Excel 支持有限，建议另存为 CSV 格式后上传。

**Q: 如何避免数据重复？**
A: 使用不同的批次 ID。系统会自动检测 MD5 重复，但不同批次 ID 可以区分数据来源。

### 审核相关问题

**Q: 为什么有些数据需要人工审核？**
A: AI 置信度 < 0.8 的数据需要人工审核，确保数据质量。

**Q: AI 自动通过的数据可以直接使用吗？**
A: 需要经过 24 小时冷却期，冷却期后自动同步到 Dify RAG 知识库。

**Q: 如何修正 AI 的系统性错误？**
A: 使用人工抽检页面的 **"批量修正"** 功能，可以按条件筛选并批量修正。

### 系统相关问题

**Q: 服务器无法启动？**
A: 检查端口 3000 是否被占用，或查看日志 `npm start` 输出。

**Q: 数据库连接失败？**
A: 检查 PostgreSQL 是否运行，以及数据库配置是否正确。

## 最佳实践

### 1. 数据上传

- **按来源分类**: 不同来源使用不同的批次 ID
- **批量上传**: 建议使用文件上传，效率更高
- **定期上传**: 建议每天固定时间上传当日数据
- **质量检查**: 上传后检查源数据列表，确认数据正确

### 2. 数据审核

- **优先处理低置信度**: 优先审核置信度 < 0.5 的数据
- **定期抽检**: 每周进行 5-10% 的抽检，监控 AI 准确率
- **批量修正**: 发现 AI 系统性错误时，及时批量修正

### 3. 批次管理

- **有意义的命名**: 使用 `来源 - 日期` 格式
- **批次追踪**: 定期检查批次处理进度
- **批次归档**: 处理完成的批次可以归档，便于后续追踪
