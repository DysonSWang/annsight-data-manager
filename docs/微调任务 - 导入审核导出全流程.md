# 微调任务 - 导入审核导出全流程指南

## 完整工作流程

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  1. 创建任务  │ ──> │  2. 导入数据  │ ──> │  3. AI 审核   │
│  配置参数    │     │  选择批次    │     │  评分 + 建议   │
└──────────────┘     └──────────────┘     └──────────────┘
                                                │
                    ┌───────────────────────────┘
                    ▼
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  6. 导出 SFT  │ <── │  5. 人工审核  │ <── │  4. AI 优化   │
│  JSONL 格式   │     │  最终确认    │     │  自动修改    │
└──────────────┘     └──────────────┘     └──────────────┘
```

---

## 步骤 1：创建微调任务

### 访问路径
`/finetuning/list.html`

### 操作步骤
1. 点击 **"+ 新建任务"** 按钮
2. 填写任务配置表单：
   - **任务名称**（必填）：如 `客服对话微调数据 -202603`
   - **微调目的说明**：描述这批数据的用途
   - **合格分值**（0-1）：默认 0.90（90 分）
   - **最多 AI 审核次数**：默认 2 次
   - **启用人工审核**：勾选后可以选择审核范围
     - 仅 AI 审核失败的数据
     - 全量数据

### 配置说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 合格分值 | 0.90 | AI 审核达到此分数视为通过 |
| 最大审核轮次 | 2 | 每条数据最多经历几次 AI 审核 + 优化 |
| 人工审核范围 | failed | all=全量，failed=仅失败数据 |

### 创建后
- 任务状态：`pending`（待开始）
- 自动跳转到任务详情页进行数据导入

---

## 步骤 2：导入数据

### 访问路径
`/finetuning/task-detail.html?id={task_id}`

### 前置条件
- 已有源数据批次（通过 `/raw-data.html` 上传或处理的数据）

### 操作步骤
1. 点击 **"导入数据"** 按钮
2. 在弹出的对话框中选择源数据批次
3. 点击 **"导入"** 按钮

### 数据关联
- 系统会将选定批次的所有 `processed_data` 关联到当前任务
- 通过 `batch_id` 字段建立关联

### 导入后
- 任务状态保持 `pending` 或变为 `importing`
- 可以在数据列表中查看导入的数据

---

## 步骤 3：AI 审核

### 访问路径
`/finetuning/task-detail.html?id={task_id}`

### 操作步骤
1. 点击 **"启动 AI 审核"** 按钮
2. 确认调用 LLM API

### 审核流程
```
每条数据
   │
   ▼
┌─────────────────┐
│ AI 审核 (第 1 轮)  │ ──> 分数 ≥ 合格线？ ──是─> ✅ 通过
│ 评分 + 建议      │          │
└─────────────────┘          否
                             │
                             ▼
                    ┌─────────────────┐
                    │ AI 优化 (第 1 轮)  │ ──> 还有剩余轮次？ ──是─> 返回 AI 审核
                    │ 根据建议修改    │          │
                    └─────────────────┘          否
                                                 │
                                                 ▼
                                        ❌ 标记为需人工介入
```

### 审核状态
- `reviewing`：审核中
- `optimizing`：优化中
- `manual_review`：人工审核中

### 进度统计
- 总数据量
- 已审核数据
- 已通过数据（≥ 合格分）
- 已失败数据（< 合格分）
- 已优化数据
- 平均分数

---

## 步骤 4：AI 优化

### 访问路径
`/finetuning/task-detail.html?id={task_id}`

### 触发条件
- AI 审核分数 < 合格分
- 还有剩余审核轮次

### 操作步骤
1. 点击 **"启动 AI 优化"** 按钮
2. 系统自动处理所有未通过的数据

### 优化流程
1. 读取 AI 审核的反馈意见
2. AI 根据建议修改数据内容
3. 更新 `processed_data` 表中的记录
4. 标记审核轮次为 `optimized`
5. 重新提交 AI 审核

---

## 步骤 5：人工审核

### 访问路径
`/finetuning/review.html?id={task_id}`

### 触发条件
- 任务启用了人工审核
- AI 审核流程完成（或手动启动）

### 操作步骤
1. 点击 **"人工审核"** 按钮
2. 进入人工审核页面
3. 逐条查看数据并决策：
   - **通过**：数据最终被接受
   - **拒绝**：数据被标记为不合格
   - **要求优化**：返回 AI 优化流程

### 人工审核范围
- `failed`：仅 AI 审核失败的数据
- `all`：所有已完成 AI 审核的数据

---

## 步骤 6：导出 SFT 数据

### 访问路径
- 列表页：`/finetuning/list.html`
- 详情页：`/finetuning/task-detail.html?id={task_id}`

### 导出条件
- 有已通过审核的数据（`review_status = 'approved'`）

### 操作步骤
1. 点击 **"导出 SFT 数据"** 按钮
2. 选择导出格式：
   - **SFT 格式**：带 `<think>` 思考标签（推荐）
   - **Messages 格式**：标准对话格式
   - **Instruction 格式**：指令 - 输入 - 输出格式
3. 选择数据范围：
   - 全部已审核通过的数据
   - 仅已通过（AI 审核 ≥ 合格线）
4. 点击 **"导出"** 下载文件

### 导出格式示例

**SFT 格式（带思考标签）**：
```json
{
  "messages": [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "如何提高职场沟通能力？"},
    {"role": "assistant", "content": "<think>\n分析沟通场景...\n</think>\n\n提高职场沟通能力可以从以下几个方面入手..."}
  ],
  "metadata": {
    "id": "pd-xxx",
    "type": "职场智慧",
    "category": "职场",
    "batch_id": "ft-batch-xxx"
  }
}
```

### 导出文件
- 格式：JSONL（每行一条 JSON）
- 文件名：`sft-export-{taskId}-{timestamp}.jsonl`
- 自动标记已导出数据（`used_in_finetuning = TRUE`）

## 完整操作示例

### 场景：创建客服对话微调数据集

**1. 创建任务**
```
任务名称：客服对话微调 -202603
微调目的：训练客服对话模型的问答能力
合格分值：0.85
最大审核轮次：2
人工审核：启用（仅失败数据）
```

**2. 导入数据**
```
选择批次：batch-20260322-001 (500 条数据)
导入结果：500 条数据关联成功
```

**3. AI 审核**
```
启动 AI 审核
并发数：10
处理结果:
  - 通过：320 条（≥85 分）
  - 失败：180 条（<85 分）
  - 平均分数：0.82
```

**4. AI 优化**
```
启动 AI 优化
并发数：5
处理结果:
  - 优化成功：180 条
  - 优化后重新审核
  - 第二轮通过：150 条
  - 仍失败：30 条
```

**5. 人工审核**
```
进入人工审核页面
审核 30 条失败数据:
  - 通过：25 条
  - 拒绝：5 条
最终通过率：98.3%
```

**6. 导出 SFT**
```
格式：SFT（带 <think> 标签）
范围：全部已通过
导出结果：495 条数据
文件大小：320 KB
文件名：sft-export-ft-xxx-1774251395.jsonl
```

## API 端点汇总

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/finetuning/task` | POST | 创建任务 |
| `/api/finetuning/task` | GET | 任务列表 |
| `/api/finetuning/task/:id` | GET | 任务详情 |
| `/api/finetuning/task/:id/import` | POST | 导入数据 |
| `/api/finetuning/task/:id/review/start` | POST | 启动 AI 审核 |

| `/api/finetuning/task/:id` | DELETE | 删除任务 |

## 常见问题

### Q1: 导入数据时提示"批次中没有数据"
**A**: 请确保：
1. 源数据批次已通过 ETL 处理
2. 数据已流入 `processed_data` 表
3. 批次 ID 正确

### Q2: AI 审核长时间未完成
**A**: 可能原因：
1. 数据量较大，正在排队处理
2. LLM API 响应慢
3. 并发数设置过低
可查看浏览器控制台或服务器日志了解进度

### Q3: 导出数据格式不符合预期
**A**: 可以尝试：
1. 切换导出格式（SFT/Messages/Instruction）
2. 检查数据是否有 `conversation` 字段
3. 查看服务器日志确认导出服务正常

### Q4: 人工审核列表为空
**A**: 可能原因：
1. 所有数据都已通过 AI 审核
2. 人工审核范围设置为"仅失败"，但没有失败数据
3. 人工审核尚未启动
